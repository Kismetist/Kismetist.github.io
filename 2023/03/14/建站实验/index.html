利用云平台搭建个人博客

本地安装

npm安装

  • 用途：安装各种插件的工具，相当于python里面的pip

  • 用法：npm install -g balabala --save

Git安装

  • 用途：链接本地文件夹和github仓库，建立push和deploy关系

Node.js安装

  • 用途：前端的语言，相当于后端中的C++

hexo安装

  • 用途：搭建博客的平台，支持各种模板

本地环境配置

安装部署插件

npm install hexo-deployer-git --save

本地文件夹初始化

cd Kismetist    # 进入文件目录
hexo init  # 博客初始化 (安装所需要的包)

本地预览

  • 这里要在博客的根目录运行
hexo s # http://localhost:4000/预览

新建文章

  • 进行设置__config.yml：

    • 这里是为了在新建md时创建同名的文件夹

  • 进行设置typora：typora-root-url: 建站实验

    • 这里是为了让图床也能自动传入文件夹中，终于没有了丢失图片的烦恼啦！
    	hexo n "第一篇博客" # 新建一个名为第一篇博客.md的文件

合个照吧！

  • 目前为止，项目的本地文件夹是这样的

  • source里是每一篇博客

  • themes是存储的主题（这个待会说）

  • _config.yml是配置文件，非常重要

Github配置

配置Git

git config --global user.name "你的Github用户名"
git config --global user.email "你的Github邮箱"

ssh-keygen -t rsa -C "你的Github邮箱"

cat ~/.ssh/id_rsa.pub

github进行确认

github博客仓库

deploy本地设置

deploy:
  type: git
  repo: git@github.com:你的用户名/你的用户名.github.io.git
  branch: main

编译与部署

hexo clean  # 清除缓存
hexo g      # 生成静态网页
hexo d      # 部署到Github

合个影吧！

  • 本地

  • 服务器

博客主题

按理来说到这里就可以结束了，但是美化之路无穷无尽，可以去网上找到很多漂亮的主题模板，这里做一个示范配置。

主题下载

  • 下载好的主题解压到themes文件夹下

本地配置

  • 设置根目录的_config.yml

  • 每一个theme都会有一个自己的_config.yml

    这是对主题的详情进行设置，例如颜色、夜间模式、link等等。

个性化支持

LatexLatexLatex数学公式

  • 由于我个人及其偏执，酷爱优美的数学公式，所以当发现我的博客板子并没有自带latexlatexlatex支持时我是崩溃的。

  • 解决办法：

    • npm install hexo-renderer-mathjax

图片插入自动化

  • 众所周知typora的图床是依托答辩，为了更加lazy的使用博客和typora，可以通过设置跟目录图床。

coding in line高亮

  • 有些根目录也会设置linghting，但可能会和主题的lighting冲突，具体如果主题有lighting设置，就在根目录的_config.yml进行来取消冲突

合个影吧！
